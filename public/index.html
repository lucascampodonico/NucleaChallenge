Descripción del Proyecto

El Proyecto es una aplicación web desarrollada con Express.js que contiene las siguientes caracteristicas:

API Endpoints

A continuación se enumeran los diferentes endpoints de tu API y una breve descripción de cada uno.

PUT /alpha

Endpoint para ordenar alfabéticamente las claves de un JSON, cargarlo en la base de datos y devolver el JSON resultante en la respuesta HTTP. 

    // Ejemplo de solicitud
    PUT /alpha
    Content-Type: application/json
    
    {
      "fruit": "apple",
      "animal": "zebra",
      "city-list": [
        "sunnyvale",
        "sanjose"
      ]
    }
    
    // Ejemplo de respuesta
    Status: 200 OK
    Content-Type: application/json
    
    {
      "animal": "zebra",
      "city-list": [
        "sunnyvale",
        "sanjose"
      ],
      "fruit": "apple"
    }

POST /flatten

Endpoint para aplanar cualquier arreglo JSON en el payload de la solicitud, de modo que el JSON resultante no contenga arreglos JSON. 

    // Ejemplo de solicitud
    POST /flatten
    Content-Type: application/json
    
    {
      "fruit": "apple",
      "animal": "zebra",
      "city-list": [
        "sunnyvale",
        "sanjose"
      ]
    }
    
    // Ejemplo de respuesta
    Status: 200 OK
    Content-Type: application/json
    
    {
      "fruit": "apple",
      "animal": "zebra",
      "city-list": "sunnyvale,sanjose"
    }

POST /quote

Endpoint para consultar una API externa y obtener una cita, luego guardarla en la base de datos y devolver el resultado. 

    // Ejemplo de solicitud
    POST /quote
    
    // Ejemplo de respuesta
    Status: 200 OK
    Content-Type: application/json
    
    {
      "author": "name",
      "id": "someid",
      "quote": "some awesome quote",
      "consultation_date": "12-03-2022"
    }

GET /quotes

Endpoint para obtener todas las citas almacenadas en la base de datos. Las citas están agrupadas por autor y ordenadas alfabéticamente. No se muestra el autor en cada cita, y dentro de cada autor, las citas se muestran en orden descendente según la fecha de consulta. 

    // Ejemplo de solicitud
    GET /quotes
    
    // Ejemplo de respuesta
    Status: 200 OK
    Content-Type: application/json
    
    {
      "Author A": [
        {
          "id": "someid",
          "quote": "some awesome quote",
          "consultation_date": "12-03-2022"
        },
        {
          "id": "someotherid",
          "quote": "some awesome quote",
          "consultation_date": "10-03-2022"
        }
      ],
      "Author B": [
        {
          "id": "someextraid",
          "quote": "some awesome quote",
          "consultation_date": "04-1-2021"
        }
      ]
    }

Requerimientos Técnicos

Debe estar programado en Node ya sea js o ts. Debe contar las siguientes características:

● Debe estar desarrollado en Express (https://expressjs.com/)
● Se conecte con una base de datos (puede ser un servicio de terceros)
● Debe implemente un logger
● Debe implemente al menos un middleware
● Debe manejar variables de entorno
● Debe implementar al menos dos test unitarios por endpoint
● Debe implementar manejo de errores
○ Rutas que no existan (404)
○ Si falla el api externa (500)
○ Validar inputs (400)
